# Paper Review Agent - 论文质量审查技能

## 概述

这是一个双代理协作的论文生成与审查系统，确保每篇论文都符合期刊格式规范和学术质量标准。

## 工作流程

```
用户请求生成论文
       │
       ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段1: 生成 (Paper Writing Agent)                          │
│  - 根据主题生成完整论文                                       │
│  - 遵循目标期刊格式                                          │
│  - 自检格式清单                                             │
└─────────────────────────────────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段2: 审查 (Paper Reviewer Agent)                          │
│  - 对照规范逐项检查                                         │
│  - 输出审查报告                                             │
│  - 判断: 通过 / 需修改 / 严重问题                           │
└─────────────────────────────────────────────────────────────┘
       │
       ▼
   ┌─────────────────────────────────────────────────────────────┐
   │  阶段3a: 审查通过                                          │
   │  - 输出最终论文                                            │
   │  - 通知用户完成                                             │
   └─────────────────────────────────────────────────────────────┘
                   │
                   │
   ┌─────────────────────────────────────────────────────────────┐
   │  阶段3b: 需修改 (循环回到阶段1)                            │
   │  - 携带审查反馈                                             │
   │  - 生成代理修订论文                                         │
   │  - 重新提交审查                                             │
   │  - 最多3次循环                                             │
   └─────────────────────────────────────────────────────────────┘
```

## PRL (Physical Review Letters) 格式规范

### 长度限制
| 项目 | 限制 | 检查方法 |
|------|------|---------|
| 正文字数 | ≤ 3,750 words | `texcount -sum paper.tex` |
| 印刷页数 | ≤ 4 pages | REVTeX two-column |
| 摘要字符 | ≤ 600 characters | 手动检查 |

### 格式要求
```
✓ 使用 revtex4-2 document class
✓ PRL摘要格式 (600字符内)
✓ 两栏布局
✓ PACS 编码
✓ 参考文献 APS 格式
✓ 图形质量 ≥ 300 DPI
✓ 作者信息和单位完整
```

### 内容要求
```
✓ 解释对广泛物理学界的重要性
✓ 创新点明确
✓ 理论框架完整
✓ 公式推导正确
✓ 图表说明清晰
```

## 审查清单 (Checklist)

### 📋 基本信息
- [ ] 论文标题
- [ ] 作者列表
- [ ] 单位信息
- [ ] 投稿期刊
- [ ] 字数统计

---

### ✅ 格式合规 (Format Compliance)

#### LaTeX 技术检查
```
[ ] LaTeX 编译无 Error（Warning 可接受）
[ ] 无 undefined citations
[ ] 无 undefined references
[ ] Float 未被 stuck（使用 [H] 或调整位置）
[ ] Overfull hbox < 10pt
[ ] Underfull hbox < 1000 badness
```

#### PRL 格式检查
```
[ ] 使用 revtex4-2 document class
[ ] 摘要 ≤ 600 字符
[ ] 正文 ≤ 3,750 words
[ ] 长度 ≤ 4 journal pages
[ ] 包含 PACS 编码
[ ] 两栏布局正确
[ ] 图形清楚，分辨率足够
```

#### 标题层级
```
[ ] 章节标题层级清晰
[ ] 无跳级（如 Introduction 后直接到 Results）
[ ] 每个章节有内容
```

---

### ✅ 学术质量 (Academic Quality)

#### 引言 (Introduction)
```
[ ] 清楚阐述问题和重要性
[ ] 文献综述完整（覆盖主要工作）
[ ] 清楚说明本文创新点
[ ] 结尾有 Roadmap 段落
```

#### 理论框架 (Theoretical Framework)
```
[ ] 理论推导完整
[ ] 公式符号定义清晰
[ ] 假设条件明确
[ ] 与前人工作对比清楚
```

#### 结果 (Results)
```
[ ] 结果描述准确
[ ] 图表与文字对应
[ ] 统计分析（如适用）
[ ] 误差分析（如适用）
```

#### 讨论 (Discussion)
```
[ ] 结果解释合理
[ ] 与前人工作对比
[ ] 局限性说明
[ ] 应用前景展望
```

#### 结论 (Conclusion)
```
[ ] 主要发现总结
[ ] 创新点重申
[ ] 未来方向建议
```

---

### ✅ 引用质量 (Citation Quality)

#### 引用完整性
```
[ ] 所有 \cite{} 都有对应 bib 条目
[ ] 所有 bib 条目都被使用
[ ] 无幻觉引用（虚假论文）
[ ] 无错误引用（信息不匹配）
```

#### 引用必要性
```
[ ] 每篇引用都与论文内容相关
[ ] 无不必要的"凑引用"
[ ] 陈述事实有引用支持
[ ] 引用分布合理（不过度集中某作者/团队）
```

#### 引用时效性
```
[ ] 有一定比例近5年文献
[ ] 经典工作被引用
[ ] 不过度依赖单一来源
```

---

### ✅ 图表质量 (Figure Quality)

#### 技术要求
```
[ ] 分辨率 ≥ 300 DPI
[ ] 字体清晰可读
[ ] 坐标轴标签完整
[ ] 图例清晰
[ ] 单位正确
```

#### 内容要求
```
[ ] 图题说明清晰
[ ] 每个图在正文中被引用
[ ] 数据来源标注（如适用）
[ ] 误差棒标注（如适用）
```

---

### ✅ 语言质量 (Language Quality)

```
[ ] 语法正确
[ ] 术语一致
[ ] 无重复表达
[ ] 逻辑连贯
[ ] 无抄袭问题
```

## 自动化配置

在 `settings.json` 中配置钩子：

```json
{
  "hooks": {
    "paperReview": {
      "autoReview": true,
      "maxIterations": 3,
      "targetJournal": "PRL",
      "saveOnPass": true
    }
  }
}
```
