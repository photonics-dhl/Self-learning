# 笔记质量审查工作流

## 概述

这是一个双代理协作的笔记生成与审查系统，确保每篇笔记都符合排版规范。

## 工作流程

```
用户请求生成笔记
       │
       ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段1: 生成 (Note Generator Agent)                          │
│  - 根据主题生成完整笔记                                       │
│  - 遵循 Obsidian排版规范                                     │
│  - 自检排版清单                                             │
└─────────────────────────────────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段2: 审查 (Note Reviewer Agent)                          │
│  - 对照规范逐项检查                                         │
│  - 输出审查报告                                             │
│  - 判断: 通过 / 需修改 / 严重问题                           │
└─────────────────────────────────────────────────────────────┘
       │
       ▼
   ┌─────────────────────────────────────────────────────────────┐
   │  阶段3a: 审查通过                                          │
   │  - 保存到目标目录                                          │
   │  - 清理 [待审查] 标记                                      │
   │  - 通知用户完成                                             │
   └─────────────────────────────────────────────────────────────┘
                   │
                   │
   ┌─────────────────────────────────────────────────────────────┐
   │  阶段3b: 需修改 (循环回到阶段1)                            │
   │  - 携带审查反馈                                             │
   │  - 生成代理修订笔记                                         │
   │  - 重新提交审查                                             │
   │  - 最多3次循环                                             │
   └─────────────────────────────────────────────────────────────┘
```

## 审查标准 (Checklist)

### 标题层级 (Heading Hierarchy)
```
[ ] H1 在笔记中只出现一次（文件开头）
[ ] 标题层级连续，无跳级（如 H2 后无 H4）
[ ] 标题简洁明确，不超过一行
```

### 文本格式 (Text Formatting)
```
[ ] 关键概念用 **加粗** 强调
[ ] 术语、英文名称用 *斜体* 标记
[ ] 高亮用于需要醒目但非关键的内容
```

### Callout 使用 (Callout Usage)
```
[ ] 物理图像使用 [!concept] 或 [!tip]
[ ] 核心公式使用 [!formula]
[ ] 关键参数使用 [!key-point]
[ ] 注意事项使用 [!warning]
[ ] 优势总结使用 [!success]
[ ] 文献引用使用 [!cite]
```

### 表格规范 (Table Standards)
```
[ ] 表头加粗
[ ] 对齐正确（数据右对齐，文字左对齐）
[ ] 有适当的单位和说明
[ ] 表格有清晰的标题行
```

### 列表格式 (List Formatting)
```
[ ] 层级缩进一致
[ ] 嵌套不超过3层
[ ] 同一列表类型连贯
```

### Mermaid 图表 (Mermaid Diagrams)
```
[ ] 图表类型选择正确（flowchart/graph/sequence/state）
[ ] 节点文字简洁（不超过20字）
[ ] 样式标注清晰（fill、stroke）
[ ] 有适当的标题或说明
```

### Frontmatter (元数据)
```
[ ] 包含 title
[ ] 包含 type (concept/paper/method)
[ ] 包含 status (studying/completed/draft)
[ ] 包含 tags
[ ] 包含 created/modified 日期
[ ] 包含 prerequisites（前置知识）
[ ] 包含 related（相关概念）
```

### 内容完整性
```
[ ] 有一句话物理图像
[ ] 有核心定义
[ ] 公式正确且完整
[ ] 参数有具体数值
[ ] 有引用文献
[ ] 有 Mermaid 知识树图
```

## 使用示例

### 用户请求
```
用户: "帮我生成太赫兹辐射源的笔记"
```

### 系统执行

**阶段1 - 生成**:
```
生成代理收到请求 → 生成初稿 → 自检 → 输出临时文件
```

**阶段2 - 审查**:
```
审查代理读取临时文件 → 逐项检查 → 输出审查报告
```

**阶段3 - 结果**:

情况A (通过):
```
笔记已保存至: Obsidian-Vault/2️⃣ 研究方向/太赫兹技术/太赫兹辐射源.md
✅ 审查通过，用时 X 秒
```

情况B (需修改):
```
⚠️ 审查发现 2 处可优化点，已反馈给生成代理
第 1 次修订中...
第 1 次修订完成，重新审查中...
```

情况C (严重问题):
```
❌ 笔记存在严重问题（公式错误/概念错误），已终止生成
原因: xxx
建议: xxx
```

## 自动化配置

在 `settings.json` 中配置钩子：

```json
{
  "hooks": {
    "noteGeneration": {
      "autoReview": true,
      "maxIterations": 3,
      "saveOnPass": true
    }
  }
}
```

## 限制与注意事项

1. **循环上限**: 最多3次生成-审查循环，避免无限循环
2. **严重问题终止**: 公式错误、概念错误等严重问题直接终止
3. **保留修改历史**: 每次修订版本保留在临时目录
4. **用户可干预**: 用户可以随时打断循环手动审查
