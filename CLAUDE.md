## Key Conventions

### Note quality requirements (mandatory)
Every note must contain: one-sentence physical intuition (callout), Ōēź1 Mermaid diagram, core formula (LaTeX), comparison table, schematic (MCP image gen or matplotlib), Ōēź2 paper citations (`[[cite:@Author2024]]`), quantified parameters.

### Knowledge deduplication
Before creating a note, run knowledge planning: scan sibling notes ŌåÆ classify relationship (`DUPLICATE`/`SUBSUMED`/`CONTAINS`/`OVERLAPS`/`INDEPENDENT`) ŌåÆ output planning card ŌåÆ then write.

### File naming (Obsidian Vault)
`NN_Title.md` or `­¤ōü NN CategoryName/`. Numbers are reading-order, not creation-order. Internal links use `[[Title]]` (no number prefix ŌĆö frontmatter `title` field handles this).

### Tag system
`#optics` / `#optics/terahertz` / `#optics/metasurface` / `#paper` / `#method`

### Language
All output, comments, docstrings in Chinese. Only physics terms, API fields, and LaTeX in English.
